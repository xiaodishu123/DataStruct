## 算法库特性

### 顺序表快速排序
- **平均时间复杂度**：O(n log n)
  - 分解：每次划分将数组分为两个子序列
  - 递归：递归深度平均为 log n
- **最坏时间复杂度**：O(n²)
  - 发生在每次划分极度不平衡时（如已排序数组且选择最左元素为基准）
  - 本实现通过取中优化减少最坏情况概率
- **空间复杂度**：O(log n)
  - 递归调用栈深度平均为 log n
  - 每层递归使用 O(1) 额外空间
- **稳定性**：不稳定排序
  - 相等元素的相对位置可能在划分过程中改变
- **原地性**：是原地排序算法
  - 只需要常数级别的额外内存空间
#### 实现细节
  本快速排序实现采用了经典的Lomuto分区方案
  
### 双向链表归并排序
- **时间复杂度**：O(n log n)
    - 分解：每层递归将链表分成两半，共 log n 层
    - 合并：每层需合并所有节点，总操作为 O(n)
- **空间复杂度**：O(log n)
    - 递归调用栈深度为 log n
    - 每层递归使用 O(1) 额外空间
- **稳定性**：稳定排序
    - 合并时相等元素保持原有顺序
- **额外优化**：减少了尾节点遍历开销
    - 在排序过程中，将尾节点临时存储在返回链表的头节点的 `prev` 指针中
    - 避免排序完成后重建循环链表时的额外O(n)遍历
- **适用性**：双向循环链表（带哨兵节点）
#### 实现细节
1. **断开循环**：排序前暂时断开循环连接，将循环链表转换为线性链表
2. **归并排序**：对线性链表执行标准的归并排序
3. **尾节点记录**：合并函数返回的链表头节点的 `prev` 指针指向该链表的尾节点
4. **重建循环**：排序后利用存储的尾节点快速重建循环链表
