## 算法库特性

### 双向链表归并排序
- **时间复杂度**：O(n log n)
    - 分解：每层递归将链表分成两半，共 log n 层
    - 合并：每层需合并所有节点，总操作为 O(n)
- **空间复杂度**：O(log n)
    - 递归调用栈深度为 log n
    - 每层递归使用 O(1) 额外空间
- **稳定性**：稳定排序
    - 合并时相等元素保持原有顺序
- **额外优化**：无尾节点遍历开销
    - 在排序过程中，将尾节点临时存储在返回链表的头节点的 `prev` 指针中
    - 避免排序完成后重新遍历链表寻找尾节点
- **适用性**：双向循环链表（带哨兵节点）
#### 实现细节
1. **断开循环**：排序前暂时断开循环连接，将循环链表转换为线性链表
2. **归并排序**：对线性链表执行标准的归并排序
3. **尾节点记录**：合并函数返回的链表头节点的 `prev` 指针指向该链表的尾节点
4. **重建循环**：排序后利用存储的尾节点快速重建循环链表
